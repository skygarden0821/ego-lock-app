# EGO LOCK — スプレッドシート連携（GAS）

非公開のスプレッドシートを**公開せずに**、必要な数値だけをアプリに渡すための中継（Google Apps Script Web アプリ）。

アプリは `settings > シート連携` に貼った `/exec` URL を叩いて、以下を自動取得する（**シートがマスター**。アプリの該当値は同期時に上書き）：

- 日次: **ライン追加数 / AD数 / 契約数**（シート①）
- **お月謝回収金額**（シート②・売上メイン）

これらは HOME 最上部の「今年の目標（月商）」「今月の目標（契約 / AD / ライン）」と、KPI・WEALTH に反映される。

---

## セットアップ

1. https://script.google.com で新規プロジェクト作成 → `Code.gs` の中身を貼り付け。
2. `CONFIG` を自分のシートに合わせて編集：
   - `sheetId` … スプレッドシートURLの `/d/` と `/edit` の間の文字列。
   - `tab` … タブ名（空なら先頭シート）。
   - `headerRow` … 見出しの行番号（通常 `1`）。
   - `dateCol` … 日付列の見出し（**部分一致**。例 `日付`）。
   - `cols` … 取りたい値の `{出力キー: 見出し}`。見出しは部分一致：
     - シート①: `lineAdds:'ライン'`, `ad:'AD'`, `contracts:'契約'`
     - シート②: `revenue:'お月謝'`（無ければ `'回収'` や `'入金'` に変更）
3. （推奨）`CONFIG.KEY` にランダムな文字列を設定。URLを知られても叩けなくなる。
4. `_test` 関数をエディタで実行 → 実行ログにJSONが出れば読み取りOK。
5. 右上 **デプロイ > 新しいデプロイ > 種類: ウェブアプリ**
   - 実行するユーザー: **自分**
   - アクセスできるユーザー: **全員**（ブラウザから読むために必須。KEYで保護）
6. 発行された `…/exec` URL をコピー。

## アプリ側

`設定 > シート連携` に：

- **GAS URL** … 上の `/exec` URL
- **連携キー** … `CONFIG.KEY` を設定したら同じ値を入力（未設定なら空）
- **自動同期** … ON で起動時・復帰時に随時取得
- **今すぐ同期** … 手動取得

## 返すJSONの形

```json
{
  "ok": true,
  "updated": "2026-07-07T09:00:00+09:00",
  "count": 42,
  "month": { "lineAdds": 120, "ad": 40, "contracts": 4, "revenue": 600000 },
  "days": {
    "2026-07-02": { "lineAdds": 70, "ad": 25, "contracts": 2, "revenue": 350000 },
    "2026-07-05": { "lineAdds": 50, "ad": 15, "contracts": 2, "revenue": 250000 }
  }
}
```

- 同日に複数行あるシートは自動で合算する。
- `DAYS_BACK`（既定180日）より古い行は返さない。

## トラブル時

- **`bad key`** … アプリの連携キーと `CONFIG.KEY` を一致させる。
- **`Failed to fetch` / CORS** … デプロイのアクセス範囲が「全員」になっているか確認。URL は `/exec` で終わること（`/dev` は不可）。
- **数値が0** … `dateCol` / `cols` の見出しがシートの実際の見出しと部分一致しているか、`tab` 名が正しいかを確認。
- シートの構成を変えたら**再デプロイ**（新バージョン）を忘れずに。
