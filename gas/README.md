# EGO LOCK — スプレッドシート連携（GAS）

非公開のスプレッドシートを**公開せずに**、必要な数値だけをアプリに渡す中継（Google Apps Script）。
Yoshiの実シート構成に合わせて `Code.gs` は設定済み。基本そのままデプロイするだけ。

## 取り込む数値（対応表）

| アプリの表示 | 取得元シート | 場所 |
|---|---|---|
| LINE追加 / AD / 契約（日次） | 「日次進捗」タブ | A〜D列（日付 / LINE追加数 / AD数 / 契約数）を日次で集計 |
| お月謝＝月商（月次） | 各月タブ `202601`〜 | 「回収」列の合計（＝支払完了の費用合計） |

これらが HOME 最上部「今年の目標（月商）」「今月の目標（契約 / AD / ライン）」に反映される。

## セットアップ手順

1. https://script.google.com →「新しいプロジェクト」→ 出ているサンプルを全消し。
2. `Code.gs` の中身を全部貼り付け。
3. コード上部の `CONFIG.KEY` に合言葉を入れる（例 `KEY: 'yoshi2026'`）。※アプリにも同じ値を入れる。
   - `sheetId` は2つとも設定済み。タブ名も「日次進捗」設定済み。基本さわらない。
4. 関数プルダウンで **`_test`** を選び ▶ 実行 → 初回は権限を許可 → 実行ログに
   `{"ok":true, "month":{...}, "monthsRevenue":{...}}` が出れば成功。
5. 右上 **デプロイ > 新しいデプロイ > 種類:ウェブアプリ**
   - 実行するユーザー: **自分**
   - アクセスできるユーザー: **全員**（ブラウザから読むため必須。KEYで保護）
6. 発行された `…/exec` URL をコピー。

## アプリ側

`記録 > 設定 > シート連携` に：
- **GAS URL**: `…/exec`
- **連携キー**: `CONFIG.KEY` と同じ文字（未設定なら空）
- **自動同期**: ON（起動時・復帰時に随時取得）
- **今すぐ同期**: 手動取得

## 返すJSON

```json
{
  "ok": true,
  "updated": "2026-07-07T09:00:00+09:00",
  "count": 551,
  "month": { "lineAdds": 11, "ad": 1, "contracts": 2, "revenue": 529900 },
  "days": { "2026-07-01": { "lineAdds": 4, "ad": 0, "contracts": 1 } },
  "monthsRevenue": { "2026-07": 529900, "2026-06": 000000 }
}
```

## つまずいたら

- **数値が0/ズレる（LINE・AD・契約）**: 「日次進捗」タブの見出しが `日付 / LINE追加数 / AD数 / 契約数` か確認。違えば `CONFIG.DAILY.cols` の右側を実際の見出しに合わせる（部分一致でOK）。
- **お月謝が0**: 各月タブ名が `202607` 形式か、「回収」列があるか確認。列名が違えば `CONFIG.OTSUKI.revenueHeader` を変更。回収列が空でも「支払完了」の費用を自動合算するフォールバックあり。
- **`bad key`**: アプリの連携キーと `CONFIG.KEY` を一致させる。
- **Failed to fetch / CORS**: デプロイのアクセス範囲を「全員」に。URL は `/exec` で終わること。
- シート構成を変えたら**新しいバージョンで再デプロイ**。
